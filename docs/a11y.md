# Accessibility audit — FootballClean

We target **WCAG 2.1 level AA**. This document captures the fixes made in
Sprint 8 and the open issues left to address before launch.

## Landmarks & structure

- Every page exposes `<header role="banner">`, `<main id="main-content">`,
  and `<footer>` so screen readers can jump between regions.
- A skip link (rendered first inside `<body>`) jumps focus to `#main-content`
  for keyboard users.
- `<main>` is `tabIndex={-1}` so the skip link can actually move focus there.
- `<nav aria-label="Primary">` distinguishes the main nav from any future
  secondary nav (footer, breadcrumbs).

## Focus management

- Global `:focus-visible` style: 2 px pitch-green ring with a dark offset for
  contrast against the dark UI.
- `MatchCard` carries a descriptive `aria-label` so each card announces both
  teams, the kick-off time (or live minute & score).
- `LiveBadge` is `role="status"` with `aria-label="Live, minute N"` so AT
  users hear the match state.

## Motion

- `prefers-reduced-motion` media query disables transitions, animations, and
  smooth scroll globally.
- `LiveBadge` animated ping has `motion-reduce:animate-none`.
- Skeleton pulse animation has `motion-reduce:animate-none`.

## Colour & contrast

- Body text on `ink-950` background uses `ink-100`/`ink-200` (≥ 12:1).
- Muted helper text uses `ink-400` (≥ 4.7:1) — still passes for normal text.
- Pitch-green CTA (`#16a34a` on `#020617`) measured at 4.9:1.

## Forms & inputs

- All `<select>` controls have an associated `<label>` — either visible or
  `aria-label` (Country / Language pickers, filters).
- `aria-busy` is set on the loading skeleton grids so AT announces "busy"
  rather than "empty".

## Internationalisation & RTL

- `<html lang>` and `dir` are set from the locale cookie before paint so
  screen readers pick the correct voice profile.
- Arabic & Urdu flip the layout via `dir="rtl"` — verified that filter
  controls, match cards, and navigation reflow correctly.

## Open items (track in a Linear/issues board)

- [ ] Bracket round labels could carry numeric round numbers to assist AT
      users on screen-readers that read in numeric order.
- [ ] Highlight card thumbnail uses `alt=""` (decorative) — confirm with a
      blind test user that the surrounding heading is sufficient context.
- [ ] Run a full screen-reader pass with VoiceOver + TalkBack and log
      findings here.
- [ ] Investigate keyboard support for the YouTube embed once it has loaded
      (currently relies on the YT iframe's own a11y).

## Tooling

- axe DevTools (browser extension) is the primary automated checker.
- Lighthouse Accessibility score is the secondary checker (target ≥ 95).
- `eslint-plugin-jsx-a11y` runs as part of `pnpm lint` — keep it green.
